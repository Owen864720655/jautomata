Input and Output

The package io package contains classes for reading from and writing to files. JAutomata supports the AT&T format for weighted and unweighted acceptors and transducers.

Reading and Loading Acceptors

Acceptors are read from and written to files using the AcceptorIO class.

Label formats

Label formats are instances of classes that implement the Format interface. They convert strings into labels and vice versa. Read methods use them to parse labels and write methods use them to format labels. JAutomata provides two very simple label formats: CharacterFormat and StringFormat, for Character labels and String labels respectively. Both formats allow the null-label to be specified in a constructor. The label format can be passed to read and write methods via the labelFormat parameter. Methods that don't have this parameter by use CharacterFormat with "--" denoting the null-label as default.

Reading and Loading Transducers

Using the following code, we can load an unweighted automaton from a file:
  1. File file = new File("filename");
  2. InputStreamReader isr = new InputStreamReader(new FileInputStream(file, "UTF-8"));
  3. Automaton<Character, Boolean> acceptor = Automata.readUnweighted(isr, "att");

The first two lines create a Reader for a file. The third line loads an unweighted acceptor from the file. The second parameter of the Automata.readUnweighted method specifies the file format. The value "att" indicates the format used by the AT&T FSM Library. Currently, this is the only format supported by JAutomata. In the future, more formats will be supported.
